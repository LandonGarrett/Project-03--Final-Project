# Project Scope Statement  
## Sustainable Homestead Resource Website

### Project Manager  
Landon Garrett

### Purpose and Justification  
The purpose of this project is to create an accessible, well‑organized website that provides practical, research‑based information on sustainable homestead management. The site will help beginners understand soil health, small‑scale livestock care, pasture management, and basic conservation practices while giving me hands‑on experience with web development and project planning.

### Project Description  
This project will produce a simple, two‑page HTML/CSS website prototype that demonstrates clear structure, consistent styling, and beginner‑friendly educational content. It reflects the principles of effective project management, including scoping, planning, risk analysis, and structured execution.

### In Scope  
- Designing and building a functional two‑page website  
- Creating original written content  
- Incorporating curated external resources  
- Publishing via GitHub Pages  
- Producing all required documentation (scope, plan, retrospective)

### Out of Scope  
- JavaScript or backend features  
- Advanced graphic design  
- Mobile‑responsive optimization beyond basic CSS  
- User accounts or interactive tools

### Deliverables  
- `index.html` homepage  
- `about.html` secondary page  
- `style.css` stylesheet  
- README.md  
- docs/scope.md  
- docs/plan.md  
- docs/retrospective.md

### Success Criteria  
- Website loads correctly  
- Navigation is clear  
- Content is accurate and well‑organized  
- Styling is consistent  
- All documents submitted on time

### Assumptions  
- GitHub Pages will function reliably  
- All content will be original or publicly available  
- Time availability remains stable  
- Only HTML/CSS required

### Constraints  
- No JavaScript  
- Must follow assignment deadlines  
- Limited time due to work and family  
- Must use free tools

### Risks  
- Time constraints  
- Technical issues with GitHub Pages  
- Scope creep  
- Balancing academic and personal responsibilities

### Stakeholders  
- Instructor (primary)  
- Class peers and potential users (secondary)  
- Project Manager: Landon Garrett
