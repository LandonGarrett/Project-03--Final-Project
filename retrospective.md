# Project Retrospective  
## Sustainable Homestead Resource Website

### What Went Well  
The project followed a clear scope, which prevented unnecessary expansion and kept the workload manageable. Using GitHub Pages early helped identify technical issues before the deadline. The planning documents created a strong roadmap that made development efficient.

### What Went Wrong  
Time constraints made it difficult to expand content depth. Balancing coursework, job responsibilities, and family obligations created pressure during the final week. Some CSS styling required more trial and error than expected.

### What I Would Do Differently  
I would begin the website prototype earlier to allow more time for refinement. I would also schedule smaller work sessions throughout the week instead of relying on longer weekend blocks. Finally, I would gather peer feedback earlier in the process to improve clarity and usability.
